# Política del repositorio

## Visibilidad

Todo repositorio nace privado por defecto.

## Descripción obligatoria

Todo repositorio debe tener una descripción clara que indique qué contiene y para qué sirve.

## Nombre

El nombre del repositorio debe usar kebab-case, sin espacios, sin tildes y sin caracteres especiales.

## Reglas

- No subir secretos.
- No subir .env reales.
- No subir node_modules.
- No subir datos reales sin autorización.
- No publicar sin checklist.
- No mezclar pruebas temporales con proyectos formales.

## Estado

Pendiente de completar al crear el remoto.
