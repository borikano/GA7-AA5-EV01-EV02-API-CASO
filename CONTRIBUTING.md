# Contribución

## Flujo de trabajo

1. Crear rama desde main.
2. Hacer cambios pequeños y claros.
3. Validar formato, pruebas y auditoría.
4. Documentar cambios relevantes.
5. Hacer commit con mensaje claro.

## Commits

Formato:

tipo(scope): descripcion

Ejemplos:

feat(auth): agregar validacion de sesion
fix(api): corregir respuesta de error
docs(readme): actualizar instalacion
security(deps): actualizar dependencia vulnerable
chore(pnpm): ajustar configuracion
